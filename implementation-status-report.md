# HsAsrDictation 当前实现情况报告

- 报告日期：2026-04-02
- 设计依据：[`design.md`](/root/proj/hs-asr/design.md)
- 设计版本标识：`design.md` 当前内容快照，SHA256 `4245d48d830943fdfd9c279f54ef6242e9fdeff0cad4ab45512d6d7930947f54`
- 说明：该 `design.md` 未内置独立版本号，本报告按当前文件内容出具。

## 结论

当前仓库已经实现了设计稿里的 MVP 主链路，覆盖托盘常驻、全局 PTT 热键、录音、模型下载与校验、本地识别、文本回写、剪贴板回退、设置页和本地日志。

和设计稿相比，当前实现的主要差异是两处技术选型偏移和若干未做完整验证的生产项：热键不是 `RegisterHotKey`，而是 `WH_KEYBOARD_LL` 低级键盘钩子；音频采集不是设计建议的 `WASAPI`，而是 `WaveInEvent`。

## 已实现内容

- 托盘常驻入口已接入，启动后不弹主界面，托盘菜单可打开设置、触发模型下载/重载、切换听写和退出。
- 全局热键已实现按下开始、释放结束的 PTT 语义，默认仍是 `Ctrl + Alt + Space`。
- 音频录制已实现，输入格式固定为 `16 kHz / mono / 16-bit PCM`，并带有 30 秒上限。
- 录音结束后会先做首尾静音裁剪，再送入 `sherpa-onnx` FunASR Nano int8 识别引擎。
- 模型可按需自动下载、解压和校验，模型目录默认落在本地用户目录。
- 文本回写已实现 `SendInput(KEYEVENTF_UNICODE)` 优先，失败后走剪贴板粘贴回退。
- 识别前会捕获前台窗口上下文，并在密码输入框场景拒绝注入。
- 设置页已支持热键、输入设备、模型目录、剪贴板回退和自动下载等本地配置。
- 本地日志已实现，日志文件按日期落盘。

## 与设计稿一致但实现方式不同

- 设计稿原先写的是 `RegisterHotKey`，当前实现改成了低级键盘钩子，以满足“按下开始、释放结束”的语义。
- 设计稿建议使用 `WASAPI`，当前实现先用 `WaveInEvent`，属于降低首版集成风险的折中。
- 设计稿中的模型包日期和当前代码一致，仍指向 `sherpa-onnx-funasr-nano-int8-2025-12-30.tar.bz2`。
