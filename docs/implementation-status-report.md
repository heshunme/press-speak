- 说明：该 `design.md` 未内置独立版本号，本报告按当前文件内容出具。

## 结论

当前仓库已经实现了设计稿里的 MVP 主链路，覆盖托盘常驻、全局 PTT 热键、录音、模型下载与校验、本地识别、文本写回、剪贴板回退、设置页、流式预览、标点后处理、通用后处理规则和本地日志。

当前实现会在最终写回前依次执行“离线标点”和“通用后处理规则”两个可开关增强层；流式结果仅用于录音期间的预览，不会持续改写目标输入框。

## 已实现内容

- 托盘常驻入口已接入，启动后不弹主界面，托盘菜单可打开设置、触发模型下载/重载、切换听写和退出。
- 全局热键已实现按下开始、释放结束的 PTT 语义，默认仍是 `Alt + Oem3`。
- 音频录制已实现，输入格式固定为 `16 kHz / mono / 16-bit PCM`，并带有 30 秒上限。
- 录音结束后会先做首尾静音裁剪，再送入 `sherpa-onnx` FunASR Nano int8 识别引擎。
- 模型可按需自动下载、解压和校验，覆盖离线、流式和标点三类模型，模型目录默认落在本地用户目录。
- 文本回写已实现 `SendInput(KEYEVENTF_UNICODE)` 优先，失败后走剪贴板粘贴回退。
- 识别前会捕获前台窗口上下文，并在密码输入框场景拒绝注入。
- 设置页已支持热键、输入设备、模型目录、识别模式、流式预览、剪贴板回退、自动下载、标点和后处理规则等本地配置。
- 设置页已扩展为页签结构，支持后处理规则列表、编辑、顺序调整、测试、保存和单条内置规则恢复默认。
- 标点处理失败时会回退原始文本，不影响主流程。
- 后处理规则系统已接入最终写回前链路，内置 `trim-whitespace`、`normalize-fullwidth-space`、`collapse-multiple-spaces` 和 `english-acronym-join` 四条默认规则。
- 用户可通过独立规则文件覆盖内置规则或新增自定义 `exact_replace` / `regex_replace` 规则。
- 本地日志已实现，日志文件按日期落盘。

## 关键实现选择

- 热键采用 `WH_KEYBOARD_LL` 低级键盘钩子，而不是 `RegisterHotKey`，以满足“按下开始、释放结束”的 PTT 语义。
- 音频采集采用 `WaveInEvent`，采样格式固定为 `16 kHz / mono / 16-bit PCM`，并通过简单裁剪满足首版录音链路。
- 标点后处理作为独立增强层存在，只作用于最终写回文本。
- 通用后处理规则也只作用于最终写回文本，不影响流式预览或录音期间的预览状态。
- `normalize-fullwidth-space` 和 `collapse-multiple-spaces` 以“内置配置规则”形式落地，而不是单独规则类，行为与设计文档目标等价。

## 尚未覆盖或未做完整验证

- 实时流式上屏未实现，当前只提供录音期间的流式预览，不会把流式结果直接持续写回目标输入框。
- 热词偏置未实现。
- ITN 显式开关未实现。
- 云同步未实现。
- 管理员窗口、远程桌面和企业 IM 输入控件的兼容性还缺少真实 Windows 机器回归测试。
- 当前运行环境不是 Windows，仍缺少真实机器上的完整 GUI 回归验证，例如托盘、热键、模型下载和文本回写链路。

## 现有验证

- 代码里已有针对静音裁剪、模型清单、模型驻留管理、标点模型供应、离线标点服务、覆盖层控制器以及后处理规则引擎/仓储/默认规则的单元测试。
- 仓库当前的自动化验证主要集中在这些可复用逻辑上；完整 Windows 桌面链路仍需要真机回归确认。
